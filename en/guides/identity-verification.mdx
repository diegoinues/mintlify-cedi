---
title: Identity Verification
---

The user accesses the **"Identity verification"** screen and selects the **"Start verification"** option to begin the process.

![Identity verification screen — start](/images/cedi-verificacion-identidad-1.png)

The system displays a QR code that the user must scan with their mobile device. Once scanned, the user accesses the provided link on their phone and selects **"Continue"** to proceed.

![QR code and continue on mobile](/images/cedi-verificacion-identidad-2.png)

Next, the system presents a form where the user must select the type of document they will use to verify their identity, choosing between **INE (voter identification card)** or **passport**, as well as the country of issuance of the document.

Subsequently, the user selects **"Take photo of the front side"**, which activates the device camera. The user must capture the image of the front of their identification, making sure to place it within the indicated frame.

The system analyzes the captured image to validate its readability and authenticity. If the document is valid, it allows continuing with the scanning of the back side; otherwise, the user is asked to repeat the capture.

Once the front has been validated, the user selects **"Take photo of the back side"**. The system activates the camera again to capture the back of the document. In the same way, the quality and validity of the image are evaluated. If the validation is successful, the user can continue; if not, they are asked to repeat the process.

After that, the user accesses the **"Get ready for the camera"** screen, where the system displays a series of recommendations to ensure a correct facial capture, such as smiling, positioning themselves in a place with good lighting, and removing any object that covers the face, including glasses.

Once ready, the user selects **"Continue"** and the camera is activated again. The user must position their face within the indicated frame to perform the capture.

The system analyzes the facial image to validate that it is clear and adequate. If the image meets the criteria, the process continues; otherwise, the user is asked to repeat the capture.

Finally, the system notifies the user that their request has been sent for review and that they will be informed later about its approval or rejection.
